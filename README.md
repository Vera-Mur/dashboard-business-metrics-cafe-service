## 📌 Задача

Построить дашборд для мониторинга ключевых продуктовых и бизнес-метрик сервиса доставки еды, чтобы отслеживать поведение пользователей, динамику заказов и эффективность привлечения.

---

## 🔍 Что сделано

- рассчитаны ключевые метрики: MAU, конверсия в заказ, ARPU  
- проанализирована воронка пользователей (просмотр → заказ → повторный заказ)  
- реализован когортный анализ пользователей  
- добавлена сегментация по каналам привлечения и платформам  
- построена динамика выручки и заказов  

---

## 📈 Ключевые выводы

- конверсия в первую покупку является узким местом воронки  
- retention пользователей снижается после первого месяца  
- отдельные каналы привлечения показывают более высокий ROI  
- наблюдается различие в поведении пользователей в зависимости от платформы  

---

## 🛠 Стек

## 📂 Структура дашборда

- динамика метрик во времени
